# 平台状态说明

更新日期：2026-06-30

Click / Sentence Reader 是一个本地优先阅读系统。它不是为每个平台各做一套读书软件，而是共用同一套书库、正文、句子、备注、红标、查词和数据保存逻辑，再给不同设备提供不同入口。

## 当前真实状态

| 平台 | 当前状态 | 说明 |
| --- | --- | --- |
| macOS | 当前主力可用 | 已有本地 App 和打包链路，是现在主要日常使用入口。 |
| iPad / 浏览器 | 当前局域网可用 | 不需要安装 App；和 Mac 在同一个 Wi-Fi 下，用浏览器打开局域网地址。 |
| Windows 浏览器版 | 计划优先支持 | 还没完成；目标是在 Windows 本机启动 Reader API 后，用浏览器打开 `/library`。 |
| Windows 桌面版 | 后续规划 | 还没完成；目标是用 WebView2 或 Tauri 做 `Click.exe`。 |
| Windows 安装包和快捷方式 | 尚未实现 | 开始菜单快捷方式、可选桌面快捷方式、卸载清理和 Windows 安装包都还是 P3 阶段工作。 |
| Windows 阅读快捷键 | Web 层已实现 | 共享 Web 阅读器已支持 `N`、`R`、`V`、`Esc`、方向键、`PageUp`、`PageDown`；未来 Windows 客户端复用这一层。 |

## 不要误解成什么

- 现在还没有完成的 Windows 安装包。
- 现在还不会创建 Windows 桌面快捷方式。
- 现在还没有完成的 Windows `Click.exe`。
- iPad / 浏览器入口不需要安装，它依赖同一局域网里的 Mac。
- macOS 是当前唯一已经可用的本地 App 客户端。

## 已经落地的 Web 阅读快捷键

这些快捷键属于 Web 阅读器和未来 Windows 路线，不是系统全局快捷键。

| 想做什么 | 快捷键 |
| --- | --- |
| 聚焦句子后添加文字备注 | `N` |
| 聚焦句子后整句标红 / 取消标红 | `R` |
| 聚焦句子后添加语音备注 | `V` |
| 关闭弹窗、抽屉或取消句子聚焦 | `Esc` |
| 翻页 | 方向键、`PageUp`、`PageDown` |

## 语音备注应该怎么说

正确文案是：

> 语音备注默认使用 Click 软件层本地识别；系统自带语音识别只是备选 / 降级路径。

在 macOS 当前版本里，这句话的意思是：

- 默认：Click 软件层本地识别，也就是配置好的 FunASR 路径。
- 备选：当软件层本地识别不可用或失败时，再降级到 Apple 系统语音识别。
- 默认不调用付费语音服务。

未来 Windows 路线也应该保持同一原则：

- 默认：Click 软件层语音路径，也就是浏览器 / WebView 录音上传到本机 Reader API，再走本地转写。
- 备选：只有 Click 本地路径不可用时，才考虑 Windows 系统或浏览器语音识别。
- 最后兜底：手动文字备注。

## Windows 什么时候才算真正完成

只有下面这些都在 Windows 上实现并验收通过，才可以说 Windows 端完成：

1. Reader API 能在 Windows 本机启动。
2. PostgreSQL 在 Windows 上可安装、可连接、可迁移。
3. `/library` 能在 Windows 浏览器打开并阅读书籍。
4. 备注、语音备注、红标、阅读位置和查词能在 Windows 上保存。
5. 共享 Web 阅读快捷键在 Windows 上通过验收。
6. `Click.exe` 能自动启动 Reader API，并在同一个窗口打开 `/library`。
7. 安装包能创建开始菜单快捷方式，并可选创建桌面快捷方式。
8. 卸载时能清理快捷方式，并清楚说明用户数据保留规则。
